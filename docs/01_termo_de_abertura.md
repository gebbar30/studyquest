# Termo de Abertura do Projeto (TAP) - StudyQuest

**1. Informações Básicas**
* **Nome do Projeto:** StudyQuest - Produtividade Gamificada com IA
* **Gerente do Projeto (Aluno):** [JHONATAS MELO DE SOUZA]
* **Sponsor (Patrocinador):** Prof. Jair Cavalcante Mendes
* **Disciplina:** Engenharia de Prompt e Vibe Coding (Unifacisa)
* **Data de Início:** [16/03/2026]

**2. Justificativa do Projeto**
Estudantes universitários enfrentam dificuldades crônicas para manter a consistência nos estudos. O projeto justifica-se pela necessidade de criar uma ferramenta que utilize técnicas modernas (Gamificação e IA) para transformar a obrigação do estudo em uma jornada engajadora, reduzindo a evasão e a procrastinação acadêmica.

**3. Escopo do MVP (O que será entregue)**
* Plataforma web responsiva (Frontend + Backend automático via Vibe Coding).
* Sistema de Autenticação de usuários.
* Motor de Inteligência Artificial para sugestão de cronogramas de estudo.
* Sistema gamificado (Ganho de XP, Níveis e Conquistas).

**4. Fora do Escopo (O que NÃO será entregue agora)**
* Aplicativo mobile nativo (iOS/Android).
* Integração financeira ou sistemas críticos.
* Redes sociais internas complexas (chat entre alunos).

**5. Cronograma de Marcos (Milestones)**
* **Fase 1 (16/03/2026):** Ideação e estruturação: Definição do problema, criação de personas com IA, propostas de valor, Lista Funcionalidades e Organização do repositorio inicial, incluindo entrega do (Termo de Abertura, Resumo Expandido).
* **Fase 2 (15/04/2026):** Arquitetura, Stack, Modelagem de Dados e Protótipo Visual.
* **Fase 3 (20/05/2026):** Construção do MVP (Front + Back) com ferramentas de IA.
* **Fase 4 (27/05/2026):** Entrega Final, Deploy e Pitch em Vídeo (5 min).

**6. Riscos Iniciais Mapeados**
* **Risco:** Dependência excessiva de ferramentas de IA gerando código com vulnerabilidades.
  * **Mitigação:** Revisão crítica humana baseada em Clean Code e testes (atuar como "Filtro").
* **Risco:** Escopo grande demais para o prazo.
  * **Mitigação:** Foco rigoroso nas "User Stories" do MVP descritas no README.
